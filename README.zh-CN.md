## 🎯 功能

**i-am** 分析你的对话历史，更新 USER.md 人格特质：

1. **加载消息** - 提取上次分析后的用户消息
2. **扎根理论** - AI 从语义提取特质（不要预定义标签）
3. **置信度追踪** - 追踪特质置信度变化
4. **更新 USER.md** - 合并到人格模型

**输出示例**：
```markdown
## 🧠 人格特质 (i-am 动态分析)

- 🔴 **沟通风格**: 直接高效 (饱和度：75%, +5%)
- 🔴 **决策风格**: 行动导向 (饱和度：68%, 不变)
- 🟡 **技术取向**: 实用主义 (饱和度：52%, -10%)
```

用户安装 skill

### 分析流程（定时/手动）

```
OpenClaw Cron (2:30 AM/PM) 或 用户触发"运行 i-am 分析"
    ↓
AI 加载上次分析后的新消息（最多 50 条）
    ↓
AI 进行扎根理论分析：
  1. 开放性编码：从每条消息提取编码
  2. 主轴编码：将编码聚类到范畴
  3. 选择性编码：提取核心特质 + 饱和度
    ↓
AI 生成预览（temp/USER.md）
    ↓
AI 发送预览文件给用户审核
    ↓
用户确认或取消
    ↓
确认：更新 USER.md + 备份到 CHANGELOG/
取消：删除 temp/USER.md
```

## 🎓 扎根理论方法

**核心原则**：不要预定义标签，从语料自然涌现！

### 三个编码阶段

| 阶段 | 输入 | 输出 |
|------|------|------|
| **开放性编码** | 用户消息 | 编码 + 范畴 |
| **主轴编码** | 开放性编码 | 范畴聚类 |
| **选择性编码** | 主轴聚类 | 核心特质 + 饱和度 |

### 饱和度计算

```
饱和度 = 0.5 + (频次 / 总消息数) * 0.5
```

### 置信度更新规则

| 场景 | 置信度计算 |
|------|-----------|
| 首次分析 | `置信度 = 饱和度` |
| 与历史一致 | `新置信度 = 旧置信度 + 0.05` |
| 与历史冲突 | `新置信度 = max(0.6, 饱和度)`（新说法权重高） |
| 用户确认 | `置信度 + 0.10`（最高 0.95） |

## ⚙️ 配置

### Cron 定时

安装时自动配置：
- **早上**: 每天 2:30 AM
- **下午**: 每天 2:30 PM

### 配置参数

```yaml
analysis:
  min_messages: 5             # 少于 5 条则跳过
  max_messages: 50            # 只处理最近 50 条
  
saturation:
  core: 0.7                   # 核心特质阈值（🔴）
  secondary: 0.5              # 次要特质阈值（🟡）
  emerging: 0.3               # 涌现特质阈值（🟢）
```
